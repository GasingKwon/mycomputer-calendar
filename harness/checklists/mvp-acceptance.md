# MVP Acceptance Checklist

## Calendar

- [ ] 앱 실행 시 현재 월 달력이 표시된다.
- [ ] 이전/다음 달 이동이 가능하다.
- [ ] 오늘 날짜가 강조된다.
- [ ] 선택한 날짜가 강조된다.
- [ ] 날짜칸에 할 일이 최대 3개 표시된다.
- [ ] 할 일이 3개를 초과하면 `+N개`가 표시된다.

## Task Management

- [ ] 날짜별 할 일을 추가할 수 있다.
- [ ] 할 일을 수정할 수 있다.
- [ ] 할 일을 삭제할 수 있다.
- [ ] 할 일을 완료/미완료로 토글할 수 있다.
- [ ] 완료된 할 일은 회색과 취소선으로 표시된다.
- [ ] 제목이 비어 있는 할 일은 저장되지 않는다.

## Task Fields

- [ ] 시간 일정을 입력할 수 있다.
- [ ] 중요도를 낮음/보통/높음으로 설정할 수 있다.
- [ ] 카테고리를 생성/수정/삭제할 수 있다.
- [ ] 할 일에 카테고리를 지정할 수 있다.
- [ ] 마감일을 입력할 수 있다.
- [ ] N일 반복을 설정할 수 있다.
- [ ] 알림 켜기/끄기를 설정할 수 있다.
- [ ] 메모를 입력할 수 있다.

## Water Tracking

- [ ] 캘린더 왼쪽에 오늘 물 마시기 컵 UI가 표시된다.
- [ ] 컵을 클릭하면 오늘 섭취량이 100ml씩 증가한다.
- [ ] 2000ml가 되면 컵이 가득 찬 상태로 표시된다.
- [ ] 물 목표를 달성한 날짜는 달력 날짜 오른쪽에 가득 찬 컵 상태 아이콘이 표시된다.
- [ ] 지난 날짜가 2000ml 미만이면 빨간 X 상태 아이콘이 표시된다.

## Recurrence

- [ ] 반복 없음 항목은 기준 날짜에만 표시된다.
- [ ] 1일 반복 항목은 기준 날짜 이후 매일 표시된다.
- [ ] 7일 반복 항목은 7일 간격으로 표시된다.
- [ ] 기준 날짜 이전에는 반복 항목이 표시되지 않는다.

## Notifications

- [ ] 시간이 있는 일정은 지정 시간에 알림이 표시된다.
- [ ] 마감일이 오늘인 작업은 알림 대상이 된다.
- [ ] 중요도 높음 작업은 알림 대상이 된다.
- [ ] 완료된 작업은 알림 대상에서 제외된다.
- [ ] 알림 꺼진 작업은 알림 대상에서 제외된다.
- [ ] Windows 기본 알림 소리가 사용된다.

## Storage

- [ ] 데이터가 로컬 JSON 파일에 저장된다.
- [ ] 앱 재실행 후 할 일 목록이 복원된다.
- [ ] 앱 재실행 후 설정이 복원된다.
- [ ] 잘못된 데이터는 앱 전체를 중단시키지 않는다.

## Window Behavior

- [ ] 창 크기 조절이 가능하다.
- [ ] 창 위치와 크기가 저장된다.
- [ ] 항상 위에 고정이 동작한다.
- [ ] 투명도 조절이 동작한다.
- [ ] 트레이 숨기기가 동작한다.
- [ ] 트레이에서 창을 다시 열 수 있다.
- [ ] Windows 시작 시 자동 실행 옵션을 켜고 끌 수 있다.

## Design

- [ ] 전체 디자인은 깔끔한 Google Calendar 계열의 정보 중심 UI다.
- [ ] 날짜칸, 상세 패널, 폼의 정보 위계가 명확하다.
- [ ] 작은 창 크기에서도 주요 기능을 사용할 수 있다.
- [ ] 완료/중요도/선택 날짜/오늘 날짜 상태가 시각적으로 구분된다.
